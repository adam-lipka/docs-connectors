Use this topic to configure a Mule application in XML from a standalone editor or using the Anypoint Studio XML editor. Studio 7 relies on Maven to provide connector dependencies.

XML and POM file configuration information, and XML-related usage topics.

== XML Namespace

If you are creating ServiceNow connector XML, add the namespace for the connector. By contrast, Studio creates the namespace for a connector automatically when you create a project for a Mule app.

[source,xml,linenums]
----
xsi:schemaLocation="
...
http://www.mulesoft.org/schema/mule/servicenow
http://www.mulesoft.org/schema/mule/servicenow/current/mule-servicenow.xsd"
----

Example of where to add these lines to your flow:

[source,xml,linenums]
----
<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:connector="http://www.mulesoft.org/schema/mule/servicenow"
      xsi:schemaLocation="
               http://www.mulesoft.org/schema/mule/core
               http://www.mulesoft.org/schema/mule/core/current/mule.xsd
               http://www.mulesoft.org/schema/mule/servicenow
               http://www.mulesoft.org/schema/mule/servicenow/current/mule-servicenow.xsd">

      <!-- put your global configuration elements and flows here -->

</mule>
----

== POM File Configuration

When creating an app manually from the XML, set up the `pom.xml` for your project:
----
<repositories>
          <repository>
            <id>anypoint-exchange</id>
            <name>Anypoint Exchange</name>
            <url>https://maven.anypoint.mulesoft.com/api/v1/maven</url>
            <layout>default</layout>
        </repository>
        <repository>
            <id>mulesoft-releases</id>
            <name>MuleSoft Releases Repository</name>
            <url>https://repository.mulesoft.org/releases/</url>
            <layout>default</layout>
        </repository>
    </repositories>
----

This snippet specifies a project dependency for a specific release version. Mule converts `RELEASE` to the released connector version when you deploy the app.

[source,xml,linenums]
----
<dependency>
  <groupId>org.mule.modules</groupId>
  <artifactId>mule-module-servicenow</artifactId>
  <version>RELEASE</version>
  <classifier>mule-plugin</classifier>
</dependency>
----

This snippet specifies a project dependency for the latest version. Mule converts `LATEST` to the released connector version when you deploy the app.

[source,xml,linenums]
----
<dependency>
  <groupId>org.mule.modules</groupId>
  <artifactId>mule-module-servicenow</artifactId>
  <version>LATEST</version>
  <classifier>mule-plugin</classifier>
</dependency>
----

Create applications using the Zuora connector. You can use an HTTP Listener or Scheduler as a trigger to initiate execution of the Mule flow.

To configure in Design Center:

. From Anypoint Platform > Design Center, create a Mule app and add a trigger such as an HTTP Listener.
. Create an HTTP element for the connector, by setting these fields:
+
[%header%autowidth.spread]
|===
|Field |Description
|Base path| Path where the Mule application listens for requests.
|Protocol | Selected for the HTTP endpoint, it can be HTTP or HTTPS (secure).
|Host| IP address where the Mule application listens for requests.
|Port| Address where the Mule application listens for requests.
|===
+
. Select the plus sign to add a component.
. Select the connector as a component.
. Select the operation.
+
To create a reusable configuration for the connector, you can use only Basic Authentication:
+
image::zuora/zuora-choose-global-type.png[Reusable configuration]
+
. Configure the Account Key for Retrieve multiple operation for the key of the account to be retrieved.
+
image::zuora/zuora-get-account.png[Design Center Get Account configuration]

=== Required Parameters for the Basic Auth Configuration

[%header%autowidth.spread]
|===
|Field |Description
|*basePath*| Base path of the REST api for your instance. Usually it is the `/rest` path.
|*Username*| Zuora username.
|*Password*| Zuora password.
|*Host*|Path to the Zuora API, for example: apisandbox-api.zuora.com.
|*Port*|Address where the Mule application listens for requests.
|===

image::zuora/zuora-api-configuration-dc.png[Zuora API Configuration]

After completing these steps, see the Examples section to experiment with the Zuora connector.
